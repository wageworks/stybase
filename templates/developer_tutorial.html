Everything you need to start using the Stybase API with your app.

OAuth 2.0 Flow
  1. Authorize User
    Redirect the user to the stybase authorization page: 
    GET /oauth/authorize
?client_id={{ app.client_id }}
&redirect_uri={{ app.redirect_uri }}
&response_type=code
&scope=read
    cURL Example: 
    curl -X GET "https://stybase.io/oauth/authorize?client_id={{ app.client_id }}&redirect_uri={{ app.redirect_uri }}&response_type=code&scope=read"
  2. Exchange Code for Access Token
    After the user authorizes, exchange the code for an access token: 
    POST /oauth/token
Content-Type: application/x-www-form-urlencoded

client_id={{ app.client_id }}
client_secret={{ app.client_secret }}
code=THE_CODE_YOU_RECEIVED
grant_type=authorization_code
redirect_uri={{ app.redirect_uri }}
    cURL Example: 
    curl -X POST "https://stybase.io/oauth/token" \
 -d "client_id={{ app.client_id }}" \
 -d "client_secret={{ app.client_secret }}" \
 -d "code=THE_CODE_YOU_RECEIVED" \
 -d "grant_type=authorization_code" \
 -d "redirect_uri={{ app.redirect_uri }}"
    Sample Response: 
    {
    "access_token": "abcd1234efgh5678",
    "refresh_token": "refresh123456",
    "expires_in": 3600,
    "token_type": "Bearer"
}
  3. Use the Access Token
    Include the token in API requests: 
    Authorization: Bearer YOUR_ACCESS_TOKEN
    Python Example: 
    import requests

headers = {"Authorization": "Bearer YOUR_ACCESS_TOKEN"}
resp = requests.get("https://stybase.io/api/userinfo", headers=headers)
print(resp.json())
    Sample Response: 
    {
    "username": "user2",
    "email": "lol@lol.lol",
    "name": "test meidiot",
    "phone": "200 19856 7890",
    "app_password": "12121212"
}
  4. Refresh Token (Optional)
    Use the refresh token to get a new access token: 
    POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=refresh_token
refresh_token=YOUR_REFRESH_TOKEN
client_id={{ app.client_id }}
client_secret={{ app.client_secret }}
    Sample Response: 
    {
    "access_token": "new_access_token_1234",
    "expires_in": 3600,
    "token_type": "Bearer"
}
  5. Revoke Authorization (Optional)
    Revoke a user's authorization if needed: 
    POST /oauth/revoke
Content-Type: application/x-www-form-urlencoded

client_id={{ app.client_id }}
user_id=USER_ID   # optional
    Sample Response: 
    {
    "success": true,
    "message": "User authorization revoked"
}
Tips & Notes
  • Keep your Client Secret private. Never expose it in the frontend.
  • Exchange authorization codes immediately as they expire quickly.
  • Always use HTTPS for API requests.
  • You can test endpoints using Postman or curl.
  • Store your access and refresh tokens securely on your server.
